# 简作 (jzPrint) 需求文档

> 版本：v1.0  
> 日期：2026-05-26

## 1. 项目背景

简作是为魔方谈定制的校本作业打印 App，面向学校场景，通过手持移动喷墨打印机将 Anoto 点阵内容打印到纸张上。学生使用 Anoto 笔在打印好的点阵纸张上书写，实现数字化采集。

### 1.1 项目来源

- 基于 TaxTicketPrinter 项目复制而来
- 设备选择、AP 连接及配网逻辑已从 inksi 中提取并独立

### 1.2 项目信息

| 属性 | 值 |
|------|-----|
| 项目名称 | 简作 |
| 包名 | com.org.jzprinter |
| compileSdk | 36 |
| minSdk | 24 |
| targetSdk | 36 |
| Java 版本 | 17 |
| Gradle 插件版本 | 8.10.1 |

## 2. 核心业务流程

### 2.1 完整流程

```
小程序分享学校信息 → 查询校本作业列表 → 获取班级/学生信息 → 下载打印素材 → 打印
```

### 2.2 小程序分享

小程序分享学校信息，格式为 JSON：

```json
{
  "schoolId": "1234567890abcd"
}
```

**重要**：小程序分享时还会携带 `businessId`，用于直接下载素材（调用 editionStudentZip 接口）。businessId 也可从 boXinRosterList 接口响应中获取。

### 2.3 查询校本作业列表

根据 `schoolId` 请求接口，返回学期及校本列表：

```json
[
  {
    "semesterId": 123,
    "semesterName": "学期名称",
    "editionList": [
      {
        "editionId": 1234,
        "editionName": "校本名称",
        "editionType": 1
      }
    ]
  }
]
```

- `editionType = 1`：指定学生铺码
- `editionType = 2`：指定预铺码

### 2.4 获取班级/学生信息（editionType = 1）

根据 `schoolId` + `editionId` + `editionType` 查询，返回班级-学生结构：

```json
[
  {
    "classId": "1234567890qwerc",
    "className": "一班",
    "businessId": "69ddf8b14588c26b3f6ffa89",
    "prepareCode": "EC100000007",
    "areaId": 1,
    "areaName": "区域名称",
    "studentList": [
      {
        "studentId": "1234567890qwerc",
        "studentName": "张三",
        "businessId": "a1b2c3d4e5f6"
      }
    ]
  }
]
```

- `StudentInfo.businessId`：班级级别的业务ID，用于下载预铺码素材
- `StudentDetail.businessId`：学生级别的业务ID，用于下载学生素材

### 2.5 获取预铺码编码（editionType = 2）

根据 `schoolId` + `editionId` + `editionType` 查询，返回预铺码列表（数据结构与2.4相同，通过 editionType=2 区分）：

```json
[
  {
    "classId": "1234567890qwerc",
    "className": "一班",
    "businessId": "69ddf8b14588c26b3f6ffa89",
    "prepareCode": "EC100000007",
    "studentList": []
  }
]
```

- `prepareCode` 为预铺码编码，`businessId` 用于下载该预铺码的素材

### 2.6 下载打印素材

- **指定学生铺码**：根据 `schoolId` + `businessId`（学生级别） + `editionType` 获取文件
- **指定预铺码**：根据 `schoolId` + `businessId`（班级级别） + `editionType` 获取文件

素材以压缩包形式下载，App 端负责解压并组织到本地目录。

**businessId 来源**：
1. 从 boXinRosterList 接口响应中获取
2. 由小程序分享直接传入（DeepLink 启动时携带）

## 3. 功能需求

### FR-01 奇偶页分离打印

**需求描述**：用户在打印时可选择打印奇数页、偶数页或全部页。

**背景**：打印本子时，奇数页（本子左侧）和偶数页（本子右侧）的纸张方向可能不同。用户手持移动打印机打印时，可能需要换方向。

**使用场景**：
- 1 人 1 台机器：先打印奇数页，调整纸张方向后，再打印偶数页
- 2 人 2 台机器：一人打印奇数页，另一人打印偶数页，并行工作

**功能要求**：
- 提供打印模式选择：全部页 / 奇数页 / 偶数页
- 奇数页 = page_1, page_3, page_5, ...
- 偶数页 = page_2, page_4, page_6, ...

### FR-02 多拼打印

**需求描述**：每页内容作为 1 拼发送给打印机打印。

**背景**：打印的内容是 Anoto 点阵。每个题目/提交码/订正码对应一张 naotu 点图（360×237 或 360×238），同一页的多个 naotu 点图无间隔纵向拼接为一张长条图（page_XX.png），这就是 1 拼。**1 页 = 1 拼 = 1 张 page_XX.png 长条图**。

**★ 打印前旋转+补白**：原图是纵向窄长条（如 360×1186），打印前旋转 90° 变为横向（1186×360），旋转后高度=360px < 打印头高度 552px，**需补白到 552px**（创建 552px 白色画布，将图放上去）。支持两种对齐方式：**顶部对齐**（靠打印头起始端）和**底部对齐**（靠打印头结束端）。补白后每页恰好 1 个 RowData = 1 拼。

**功能要求**：
- 1 页 = 1 拼 = 1 张纵向拼接的长条图（page_XX.png）
- 打印前：旋转 90°（纵向→横向）+ 补白到打印头高度 552px
- 支持顶部对齐 / 底部对齐两种放置方式
- 支持顺时针90° / 逆时针90° 两种旋转方向（可配置）
- 补白后每页恰好 1 个 RowData，`onPrintComplete` 每回调 1 次 = 完成 1 页
- `targetPages[currentIndex]` 直接映射为当前完成的页码

### FR-03 素材两种使用模式

**需求描述**：支持两种素材组织方式，按需选择。

**模式说明**：
- **直接使用模式**：使用已拼接好的 `page_XX.png` 长图直接打印（快速、按原样）
- **自定义拼接模式**：从 `page_XX/` 子目录读取单点阵图，自行纵向拼接后打印（支持题目重组、筛选打印）

**素材结构示例**：
```
pages/
├── page_85.png (360x1186)         ← 已拼接好的完整页
├── page_85/                        ← 单点阵图目录
│   ├── 1.png (360x237)             ← 题目1点阵
│   ├── 2.png (360x237)             ← 题目2点阵
│   ├── 3.png (360x237)             ← 题目3点阵
│   ├── 提交码.png                  ← 提交码点阵
│   └── 订正码.png                  ← 订正码点阵
├── page_86.png
├── page_86/
│   └── ...
└── ...
```

### FR-04 素材下载与管理

**需求描述**：通过客户提供的 API 下载素材压缩包，本地解压后用于打印。

> **⏳ API 待定**：客户 API 接口尚未提供。开发阶段使用 Mock 数据，待拿到实际 API 后接入。

**功能要求**：
- 从 API 下载素材压缩包（.zip）
- 本地解压到指定目录
- 不同学生/预铺码的素材独立存储，互不覆盖
- 已下载的素材不重复下载（缓存检查）
- 支持素材空间管理，打印完成后可选择性清理释放空间

### FR-05 断点续打

**需求描述**：支持中途离开后继续打印，每次打印完 1 拼（1 页）实时记录进度。

**背景**：打印任务可能包含大量页面，中途用户可能需要离开，下次打开 App 时应能继续打印。

**功能要求**：
- 每完成 1 拼（1 页），实时将当前页标记为已打印并持久化保存
- 打开 App 时，检测该学生/预铺码是否有未完成的打印任务
- 如有未完成任务，提示用户是否继续打印
- 继续打印时，从上次断点处恢复，跳过已打印的页面
- SDK `OnPrintListener.onPrintComplete()` 回调触发时记录该拼完成

### FR-06 数据持久化

**需求描述**：下载的素材、学生信息、打印进度等需要持久化保存到本地数据库。

**功能要求**：
- 学生信息缓存（离线可用）
- 素材下载状态记录
- 打印任务及进度记录
- 预铺码信息缓存

### FR-07 多学生多任务切换

**需求描述**：支持不同学生交替打印，各自进度独立保存和恢复。

**使用场景**：
- 张三打印到一半，老师叫去给李四打印
- 给李四打完后，回来继续张三的
- 甚至张三、李四、王五交替打印

**功能要求**：
- 切换学生时，当前打印任务自动暂停（PAUSED），进度不丢失
- 每个学生 + 每个校本 + 每个打印模式 = 独立的 PrintTask
- 主界面/任务列表展示所有可续打任务
- 选择任一任务即可继续打印，无需重新下载素材或重新选择配置
- 已完成任务的素材可选择性清理

**★ 手持移动打印机特性说明**：
- 打印分两阶段：数据传输（App→打印机内存）和物理打印（打印机内存→纸面）
- 协议无 StopPrint 指令，暂停时当前批次数据会打印完毕，`onPrintComplete` 继续触发推进 `printedPages` 是正确的
- 暂停 = "当前批次打完后，不再发起新打印任务"
- 切换学生发送新数据时，**打印机缓冲区中旧数据被覆盖**，旧学生未打印页的数据丢失
- `printedPages` 记录的是"哪些页已物理打印完成"，回来后续打时据此计算 remaining 并重新发送未打印页的数据

### FR-08 重打指定页

**需求描述**：打印完成后，如果某些页面打错了（模糊、偏移、缺行等），支持选择指定页重新打印。

**使用场景**：
- 发现 page_87 打歪了，只重打这一页
- 发现 page_91、page_95 有问题，选中两页一起重打
- 整本质量不满意，全部重打

**功能要求**：
- 已完成任务的打印进度界面，支持选择已完成的页面进行重打
- 支持单选和多选页码
- 重打完成后，页码重新记入 `printedPages`
- 全部重打时，清空 `printedPages`，从头开始

**重打模式**：

| 模式 | 说明 | 当前 | 未来 |
|------|------|------|------|
| 重新发送模式 | App 重新加载素材并完整发送数据给打印机 | ✅ 可用 | ✅ 始终可用（兜底） |
| 指令重打模式 | App 发送"重打第N拼"指令，打印机从缓冲区重打 | ❌ 暂不可用 | ⏳ 需打印机固件+SDK配合 |

- 当前实现重新发送模式，功能完整
- 设计上预留 `ReprintStrategy` 抽象层，未来可无缝切换为指令重打
- 指令重打需打印机固件支持缓冲区保留 + SDK 新增 `sendReprintCommand()` 接口

### FR-09 双人双机协作

**需求描述**：支持 2 人 2 台机器并行打印同一学生的作业。

**使用场景**：
- 设备 A：选择"奇数页"模式，打印所有奇数页
- 设备 B：选择"偶数页"模式，打印所有偶数页

**功能要求**：
- 两台设备各自维护独立的打印任务
- 奇偶页分离确保不重复、不遗漏
- 各自的打印进度独立记录

## 4. 非功能需求

### NFR-01 数据可靠性

- 打印进度每完成 1 拼必须持久化，避免因异常退出丢失进度
- 素材下载需校验完整性（文件大小/MD5）
- 数据库操作需事务保证

### NFR-02 存储管理

- 素材压缩包解压后可删除原 zip 文件释放空间
- 提供存储空间使用情况查询
- 打印完成的素材可按策略清理

### NFR-03 离线支持

- 已下载的素材和学生信息应支持离线访问
- 断网情况下可继续打印已下载的素材
- 重新联网后同步最新数据

### NFR-04 用户体验

- 打印进度实时展示（当前第几页/共几页）
- 断点续打提示清晰
- 打印异常时给出明确错误信息

## 5. 约束与假设

### 5.1 约束

- 打印内容为 Anoto 点阵，不是普通文字或图形
- 每个 naotu 点图宽度固定 360px，高度约 237-238px
- 1 页 = 1 拼 = 1 张 page_XX.png 长条图（多个 naotu 点图无间隔纵向拼接）
- 打印前旋转 90° + 补白到打印头高度 552px：纵向(360×H) → 横向(H×552)，原图放在白色画布上
- 支持顶部对齐 / 底部对齐两种放置方式（由用户设置选择）
- 补白后每页恰好 1 个 RowData = 1 拼，`onPrintComplete` 每回调 1 次 = 完成 1 页
- 使用 mxSdk 中已有的 MultiRow 打印机制
- 素材来源为远程 API 下载，非本地预置

### 5.2 假设

- 素材压缩包内的目录结构遵循 `page_XX.png` + `page_XX/` 的约定
- `page_XX.png` 是 `page_XX/` 目录下所有 naotu 点图无间隔纵向拼接的结果
- API 返回的素材可通过 schoolId + businessId + editionType 唯一标识
- businessId 可从 boXinRosterList 响应获取，也可由小程序分享传入
- SDK `OnPrintListener` 回调可靠，1页=1拼，`currentIndex` 直接对应页码索引
- 打印机连接状态由 mxSdk 管理，App 层不处理底层连接

## 6. 术语定义

| 术语 | 定义 |
|------|------|
| Anoto 点阵 | 一种特殊的微点图案，Anoto 笔可在其上书写并数字化采集 |
| 拼 | 打印的独立单元，1 拼 = 1 页 = 1 张 page_XX.png 长条图（旋转+补白后恰好 1 个 RowData） |
| RowData | SDK 按 552px 高度拆分的打印行数据，补白后每页恰好 1 个 RowData |
| MultiRow | SDK 的多拼打印机制，支持一次发送多拼数据 |
| 铺码 | 指定学生打印时使用的编码方案 |
| 预铺码 | 预设的打印编码，不绑定特定学生 |
| editionType | 校本作业类型，1=指定学生铺码，2=指定预铺码 |
